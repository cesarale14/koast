koast brand · motion philosophy exploration

Idle motion vs active motion.

If the banded-O is both the brand mark and the thinking indicator, motion has to do real work distinguishing "Koast at rest" from "Koast working." Three philosophies, evaluated honestly. Output: a recommendation, with implementation guidance for every deployment surface.

v0.1 · 2026.04.30 · blocks production deliverables until decision is made
A

Quiet idle, distinct active

Two distinct motion registers. Idle is near-imperceptible — a 2% opacity breath over 8s, just enough to feel "alive" without demanding attention. Active is the deposit gesture at faster tempo, clearly visible while processing. The transition between states is itself the signal.

CSS specification

/* IDLE — whole-mark breath */
@keyframes koast-breath {
  0%, 100% { opacity: 1; }
  50%      { opacity: 0.98; }
}
.koast-mark[data-state="idle"] svg {
  animation: koast-breath 8s infinite ease-in-out;
}

/* ACTIVE — dramatic deposit, 1.8s cycle */
@keyframes koast-ghost-active {
  0%, 30%, 100% { transform: translateY(0);    opacity: 0; }
  55%, 75%      { transform: translateY(23px); opacity: 1; }
}
@keyframes koast-stack-active {
  0%, 30%, 100% { transform: translateY(0);    }
  55%, 75%      { transform: translateY(18px); }
}
.koast-mark[data-state="active"] .ghost {
  animation: koast-ghost-active 1.8s infinite cubic-bezier(0.45, 0, 0.55, 1);
}
.koast-mark[data-state="active"] .stack {
  animation: koast-stack-active 1.8s infinite cubic-bezier(0.45, 0, 0.55, 1);
}

strengths

  • Most refined of the three — Goldilocks balance
  • Clear behavioral distinction between idle and active
  • Mark feels "alive" without demanding attention
  • Two motion registers serve different functions cleanly

risks

  • 2% opacity breath may be invisible to most users → A collapses into C functionally
  • Most complex to implement (two animation systems, state machine, transitions)
  • Subtle idle motion is fragile across browsers and rendering pipelines
  • If you have to explain the breath, it's not working
B

Continuous motion, varying tempo

Always animating. The mark is never still. Activity is signaled by speed change (slow → fast) plus a saturation lift on the colors. The motion is the brand presence. Same gesture, different tempo and intensity per state.

CSS specification

/* PASSIVE — always-on gentle deposit */
@keyframes koast-ghost-passive {
  0%, 70%, 100% { transform: translateY(0);    opacity: 0; }
  82%, 90%      { transform: translateY(23px); opacity: 1; }
}
@keyframes koast-stack-passive {
  0%, 70%, 100% { transform: translateY(0);   }
  82%, 90%      { transform: translateY(3px); }
}
.koast-mark .ghost { animation: koast-ghost-passive 4s infinite cubic-bezier(0.45, 0, 0.55, 1); }
.koast-mark .stack { animation: koast-stack-passive 4s infinite cubic-bezier(0.45, 0, 0.55, 1); }

/* ACTIVE — same gesture, faster + saturation lift */
.koast-mark[data-state="active"] {
  filter: saturate(1.18);
}
.koast-mark[data-state="active"] .ghost {
  animation: koast-ghost-passive 2s infinite cubic-bezier(0.45, 0, 0.55, 1);
}
.koast-mark[data-state="active"] .stack {
  animation: koast-stack-passive 2s infinite cubic-bezier(0.45, 0, 0.55, 1);
}

strengths

  • Mark always feels "alive" — strong personality
  • Single animation system (one gesture, two tempos) — simpler than A
  • Speed-change is a learnable signal users can internalize
  • Saturation lift adds a second axis of distinction

risks

  • Ambient motion in B2B dashboard is fatiguing — users staring for hours will tire of it
  • Animation fatigue: people stop noticing motion they always see
  • Battery + accessibility costs (especially in mobile contexts)
  • Speed-change is a subtler activity signal than presence/absence
  • Most successful brand marks (Stripe, Linear, Vercel, Anthropic) are static
  • Risk of being read as "loading forever" or "broken"
C

Static at rest, dramatic active

No motion at rest. Period. Active state is the full deposit gesture — clearly visible, distinct moment of activity. The cleanest semantic split of the three: motion = activity, no motion = at rest. Most defensible on attention, accessibility, and convention grounds.

CSS specification

/* IDLE — no animation. Mark is static. */
.koast-mark[data-state="idle"] {
  /* nothing */
}

/* ACTIVE — dramatic deposit, 3s cycle, 50% rest */
@keyframes koast-ghost-active {
  0%, 50%, 100% { transform: translateY(0);    opacity: 0; }
  70%, 85%      { transform: translateY(23px); opacity: 1; }
}
@keyframes koast-stack-active {
  0%, 50%, 100% { transform: translateY(0);    }
  70%, 85%      { transform: translateY(18px); }
}
.koast-mark[data-state="active"] .ghost {
  animation: koast-ghost-active 3s infinite cubic-bezier(0.45, 0, 0.55, 1);
}
.koast-mark[data-state="active"] .stack {
  animation: koast-stack-active 3s infinite cubic-bezier(0.45, 0, 0.55, 1);
}

strengths

  • Cleanest semantic split: motion = activity, no motion = rest
  • Aligns with industry convention (Stripe, Linear, Vercel, Anthropic)
  • Battery-friendly, accessibility-friendly
  • Doesn't compete with content for attention
  • Animation when present has more meaning
  • Easiest to implement and maintain
  • Most defensible on every dimension

risks

  • "Feels dead at rest" — but only if users expect motion (most don't)
  • Loses some "personality" if motion was the brand's defining trait
  • Less distinct from competitors who also use static marks
  • The mark needs to carry brand voltage on its own (which the strata gradient does)

Recommendation

Philosophy C, with one context-specific exception

C in app   B-passive on marketing hero   A reserved for future

Why C is the strongest default: Industry convention favors static marks. Motion when present has more semantic meaning. In a B2B dashboard where users spend hours, attention conservation matters more than ambient personality. The brand metaphor (accumulated strata) doesn't need constant motion to communicate — the static bands already say "deposited over time." Reserving the deposit gesture for active states strengthens its meaning rather than diluting it.

Why the exception: The public marketing landing hero is a "wow moment" surface — first-time visitors, no expectation of staying long, motion serves discovery rather than fatigue. Philosophy B's slow ambient deposit (4s cycle, 70% rest) earns its place there. Treat this as a single-context exception, not a pattern.

Why not A (yet): The 2% breath risks being invisible to most users, in which case A becomes C with extra implementation cost. If a future deployment surface needs "presence-without-attention" specifically (e.g. an always-visible app icon in a system tray, an omnibar, a passive widget), revisit A then. For now, simpler is better.

Implementation guidance — by context

contextstatespec
App header / chrome STATIC always No animation. The mark is the brand mark, not a status indicator.
In-app Koast avatar (chat bubble) STATIC unless active Static when message is settled. Active deposit (3s cycle) while the AI is generating, like Claude/Cursor's "thinking" indicator.
Loading / processing states ACTIVE 3s cycle deposit. Replaces or augments traditional spinners — the gesture conveys both "working" and brand identity.
Sync / pricing engine running ACTIVE while running 3s cycle deposit. Stops when sync completes. Static otherwise.
Browser tab favicon STATIC always Animated favicons are universally annoying. 3-band static, transparent bg.
App icon (iOS, Android, Windows tile) STATIC always Platforms forbid animated app icons anyway.
Marketing site — landing hero AMBIENT B-passive: 4s cycle gentle deposit, always on. Earns attention from new visitors.
Marketing site — body / pricing / docs STATIC No motion outside the hero. Don't make the marketing site noisy.
Email signatures, pitch decks, social cards STATIC Doesn't matter — static contexts. Use SVG masters as-is.
Onboarding / empty states ACTIVE for "first deposit" moment When user completes their first action (connects iCal, ships first listing), trigger one full deposit cycle as a celebration moment. Then static.

Reduced-motion compliance

@media (prefers-reduced-motion: reduce) {
  .koast-mark .ghost,
  .koast-mark .stack {
    animation: none !important;
  }
}

All Koast deployments must honor prefers-reduced-motion: reduce. When users opt out of motion, the mark stays static in active states too — replace the deposit animation with a one-shot opacity fade or static "loading" indicator if a status signal is required.
